## 1. 模块职责

自动化脚本引擎（`automation`）是系统的执行机构，用于远程排查和修复 Windows 终端故障。由于该模块具有强大的系统变更能力（WinRM 以 Administrator 或 SYSTEM 权限执行），因此必须建立严格的**安全审计边界、参数校验规则以及高风险二次确认流程**，防止因恶意指令或 AI 误判定导致大规模桌面系统损毁。

---

## 2. 内置脚本库规格

系统内置一套针对常见桌面故障的 PowerShell 诊断/修复脚本：

| 脚本标识 | 名称 | 风险等级 | 参数输入 | 动作描述 |
|---------|------|----------|---------|---------|
| `network-reset` | 网络配置重置 | **Medium** | 无 | 释放/续租 IP、清除 DNS 缓存、重置 Winsock，重启网卡 (造成短暂断网) |
| `office-repair` | Office 快速修复 | **Low** | 无 | 重启 Office 服务，调用 `OfficeClickToRun.exe` 触发联机/快速修复 |
| `cache-cleanup` | 系统垃圾清理 | **Low** | `target_dirs` (列表) | 清理 Windows 临时目录、用户 Temp 目录、浏览器缓存及 Windows Update 下载目录 |
| `printer-reset` | 打印机状态重置 | **Low** | `printer_name` (字符串) | 重启 `Spooler` 打印服务，清空 C:\Windows\System32\spool\PRINTERS 队列 |
| `drive-remap` | 网络共享盘挂载 | **Medium** | `share_path` (字符串), `drive_letter` (单个字母) | 校验 SMB 连接性，清理断开的映射并重新挂载网络路径为指定盘符 |
| `uninstall-software` | 软件静默卸载 | **High** | `product_id` (UUID/GUID) | 调用 MSIEXEC 或软件原生卸载程序在后台执行静默卸载 |
| `os-reboot` | 系统强制重启 | **High** | `delay_seconds` (整数) | 发送系统关机/重启命令，并在用户屏幕弹出气泡通知提示 |

## 3. 参数验证与输入清洗

为防止命令注入（Command Injection，例如参数传入 `; Remove-Item C:\*`），所有由 AI 或工程师发起的脚本执行请求必须强制进行参数 schema 校验：

### 3.1 参数 JSON Schema 校验

在 Celery 调度任务之前，使用 `jsonschema` 库验证输入参数。例如 `drive-remap` 脚本参数定义：

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "share_path": {
      "type": "string",
      "pattern": "^\\\\\\\\[a-zA-Z0-9_.-]+\\\\[a-zA-Z0-9_$.-]+(\\\\[a-zA-Z0-9_$.-]+)*$"
    },
    "drive_letter": {
      "type": "string",
      "pattern": "^[a-zA-Z]$"
    }
  },
  "required": ["share_path", "drive_letter"],
  "additionalProperties": false
}
```

### 3.2 敏感参数脱敏

在将执行日志存储到数据库 `automation_executions` 表或输出到前端控制台时，必须通过正则表达式过滤日志中可能包含的**明文密码、API 密钥以及 Session 令牌**，将其替换为 `[REDACTED]`。

## 4. 脚本执行沙箱与安全策略

1.  **进程级别权限隔离**：Agent 在 Windows 端运行时，可配置以特定受限低权限账户（例如 `IT-Service-Helper`）而非全局 `SYSTEM` 账户创建进程。对于只读诊断操作，使用受限组策略运行。
2.  **执行路径白名单限制**：只允许在自研 Agent 专属目录（如 `C:\Program Files\IntelliServe Agent\temp\`）下生成临时 `.ps1` 文件并执行，防止对系统盘敏感目录的非授权写入。
3.  **超时断开机制**：所有的脚本执行强制配置超时阈值（通常为 60s - 300s）。如果脚本在规定时间内未退出（例如出现弹窗卡死或死循环），Celery Task 将调用 Win32 API `TerminateProcess` 强行终止目标进程，避免资源耗尽。
4.  **禁用敏感 Cmdlets**：在生产环境中，Agent 的 PowerShell 会话可设置为 "Constrained Language Mode"，以禁用反射与敏感的 Win32 API 交互。

## 5. 审批工作流与双因子认证

脚本触发按风险等级（`risk_level`）走不同的控制流：

```
                    ┌────────────────────────┐
                    │  发出脚本执行请求      │
                    └───────────┬────────────┘
                                │
                  ┌─────────────┴─────────────┐
                  │                           │
                  ▼ (Low 风险)                 ▼ (Medium / High 风险)
        ┌──────────────────┐        ┌──────────────────┐
        │   自动后台执行   │        │     意图判定     │
        └──────────────────┘        └─────────┬────────┘
                                              │
                                              ▼
                                    ┌──────────────────┐
                                    │    推送审批流    │
                                    │  (钉钉消息卡片)  │
                                    └─────────┬────────┘
                                              │
                                  ┌───────────┴───────────┐
                                  │                       │
                                  ▼ (未通过/超时)         ▼ (管理员点击同意)
                        ┌──────────────────┐    ┌──────────────────┐
                        │   拒绝并记录审计 │    │  下发 Agent 执行 │
                        └──────────────────┘    └──────────────────┘
```

*   **Medium 风险**：用户自助服务触发时，需要用户在客户端或聊天卡片上点击 "确认同意" 按钮方可执行。
*   **High 风险**：必须由 IT 工程师或采购管理员审批。审批消息通过钉钉 Stream Mode 实发审批人，卡片内含脚本明细、目标 PC、风险原因以及 "同意/拒绝" 的即时按钮。对于极高风险（如批量执行命令），审批人需通过双因子验证（MFA 输入动态口令）。
*   **黑名单机制**：禁止在晚上 22:00 至次日 7:00 期间自动执行 Medium/High 级脚本，避免在非工作时间造成误关机或网络中断故障。

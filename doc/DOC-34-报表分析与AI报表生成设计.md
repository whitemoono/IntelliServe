## 1. 设计目标

报表分析页面面向 IT 管理员、运维经理、财务/合规负责人，目标不是只展示图表，而是形成“数据汇总 → 异常解释 → AI 建议 → 人工确认动作 → 审计留痕”的管理闭环。

第一阶段以静态规则和可解释指标为主，AI 负责生成摘要、发现、解释和建议，不直接执行调拨、分配 IP、卸载软件、报废资产或提交预算。

## 2. 报表页面总体设计

### 2.1 页面结构

| 区域 | 设计内容 |
|------|----------|
| 顶部筛选区 | 统计周期、部门、资产类型、园区/VLAN、负责人、报表类型 |
| KPI 概览 | 当前值、同比、环比、风险等级、目标差距 |
| 趋势图 | 日/周/月趋势，支持同比和环比切换 |
| 明细表 | 支持排序、筛选、导出和跳转到资产/工单/许可详情 |
| AI 结论 | 摘要、关键发现、原因解释、影响范围、建议动作 |
| 待确认动作 | 创建工单、提交审批、导出报表、发送周报、生成整改清单 |
| 导出区 | PDF、Excel、CSV，后续可支持钉钉/邮件推送 |

### 2.2 报表分类

| 报表类型 | 使用角色 | 核心问题 |
|----------|----------|----------|
| 工单效率报表 | 运维经理 | 工单是否及时处理，瓶颈在哪 |
| IT 成本报表 | IT 管理员、财务 | 成本花在哪，是否超预算 |
| 合规安全报表 | 合规负责人、安全管理员 | 软件和安全策略是否违规 |
| 资产利用率报表 | IT 管理员 | 哪些资产低利用、可盘活 |
| 资产/IP/预算报表 | 运维经理、财务 | IP 是否够用，资产是否该换新 |
| AI 自定义报表 | 管理层、IT 负责人 | 用自然语言生成专题分析 |

## 3. 报表明细设计

### 3.1 工单效率报表

| 模块 | 内容 |
|------|------|
| KPI | 工单总数、解决率、平均响应时长、平均解决时长、SLA 达成率、AI 自动解决率 |
| 趋势 | 按日/周展示创建、解决、积压工单趋势 |
| 维度 | 部门、工程师、故障类别、优先级、来源渠道 |
| 明细 | 工单号、标题、类别、优先级、处理人、响应时长、解决时长、满意度 |
| AI 结论 | 识别高频问题、处理瓶颈、工程师负载不均、可自动化场景 |

AI 建议示例：

- “网络类工单占比 36%，其中 DNS/网关问题重复率高，建议沉淀为 L1 自动化脚本。”
- “李工本周高优先级工单平均解决时长高于团队 42%，建议临时分流打印机类工单。”

### 3.2 IT 成本报表

| 模块 | 内容 |
|------|------|
| KPI | 年度预算、已发生费用、预计全年费用、资产折旧、维修成本、许可费用 |
| 趋势 | 月度采购、维修、订阅续费趋势 |
| 维度 | 部门、资产类别、供应商、采购批次、成本类型 |
| 明细 | 资产编号、购入金额、购入日期、账面净值、维修成本、预计残值 |
| AI 结论 | 识别超预算部门、高维修成本资产、可盘活资产节省金额 |

### 3.3 合规安全报表

| 模块 | 内容 |
|------|------|
| KPI | 合规率、违规软件数量、许可超额数量、补丁合规率、加密覆盖率 |
| 趋势 | 违规数量、整改完成率、补丁覆盖率趋势 |
| 维度 | 部门、软件、资产类型、风险等级、整改状态 |
| 明细 | 资产、软件、版本、违规原因、负责人、整改建议、处理状态 |
| AI 结论 | 识别违规软件集中部门、闲置许可、需采购或卸载的软件 |

AI 不提供破解、绕过授权、隐藏审计记录等操作建议。

### 3.4 资产利用率报表

| 模块 | 内容 |
|------|------|
| KPI | 在线率、闲置资产数、低利用资产数、可盘活金额、健康度均值 |
| 趋势 | 资产在线率、闲置数量、健康风险趋势 |
| 维度 | 部门、资产类型、型号、购入年份、使用人 |
| 明细 | 资产编号、使用率、在线时长、闲置评分、健康度、调拨建议 |
| AI 结论 | 识别可调拨设备、长期离线资产、折旧到期但健康良好的备机 |

### 3.5 资产/IP/预算报表

| 模块 | 内容 |
|------|------|
| KPI | 盘点差异、IP 冲突、空闲 IP、可盘活资产、换新预算、预计节省 |
| 趋势 | IP 使用率、盘点差异、换新预算月度趋势 |
| 维度 | VLAN、地址池、部门、资产类别、预算归属 |
| 明细 | 差异类型、IP 地址、MAC、资产、预算金额、建议动作 |
| AI 结论 | 识别 IP 紧张网段、冲突地址、可回收 IP、换新优先级 |

### 3.6 AI 自定义报表

AI 自定义报表支持管理员用自然语言提出分析需求，例如：

- “生成本月各部门 IT 成本和闲置资产节省建议。”
- “找出过去 30 天打印机故障高发的部门和型号。”
- “统计研发部终端软件合规风险，并生成整改清单。”

系统需将自然语言需求转为结构化生成条件，展示给用户确认后再生成报表。

## 4. 数据来源与指标口径

### 4.1 数据来源

| 数据源 | 用途 |
|--------|------|
| 资产台账 | 资产类别、部门、使用人、购入成本、折旧、状态 |
| 工单数据 | 创建时间、响应时间、解决时间、分类、优先级、满意度 |
| 监控数据 | 在线状态、CPU/内存/磁盘、健康度、告警 |
| 软件许可 | 许可数量、使用数量、到期时间、合规状态 |
| IPAM | 地址池、IP 分配、冲突、保留、MAC/IP 绑定 |
| 盘点差异 | 台账缺失、网络未知、IP/MAC 不一致、使用人不一致 |
| 自动化执行日志 | 脚本执行次数、成功率、节省时间、失败原因 |
| 审计日志 | 高风险动作、审批、导出、报表生成记录 |

### 4.2 指标口径

| 指标 | 计算口径 |
|------|----------|
| 解决率 | 已解决/已关闭工单数 ÷ 工单总数 |
| 平均响应时长 | 首次分派或首次回复时间 - 创建时间 |
| 平均解决时长 | 解决时间 - 创建时间 |
| AI 自动解决率 | AI 自动解决工单数 ÷ 可自动化工单数 |
| 资产在线率 | 周期内有心跳资产数 ÷ 应在线资产数 |
| 闲置评分 | 离线天数、登录频率、性能活动、网络活动、人工状态加权 |
| 账面净值 | 购入金额 - 已计提折旧，最低不低于残值 |
| 可盘活金额 | 可调拨资产的替代采购金额或账面净值估算 |
| IP 使用率 | 已分配地址数 ÷ 可用地址总数 |
| 软件合规率 | 合规安装数 ÷ 受控软件安装总数 |

### 4.3 风险等级

| 等级 | 条件示例 | 展示方式 |
|------|----------|----------|
| 低 | 指标正常，趋势稳定 | 绿色标签 |
| 中 | 指标偏离阈值但未影响核心服务 | 黄色标签 |
| 高 | 影响业务或需要管理员处理 | 红色标签 |
| 严重 | 存在合规、预算或大范围服务风险 | 红色高亮并建议创建任务 |

## 5. AI 报表生成条件

### 5.1 触发方式

| 触发方式 | 触发条件 | 典型报表 |
|----------|----------|----------|
| 手动生成 | 用户点击“生成 AI 报表” | 自定义专题、临时报表 |
| 定时生成 | 每周/每月固定时间 | 周报、月报、预算报表 |
| 异常触发 | 指标超过阈值 | IP 冲突、许可超额、SLA 下降 |
| 审批触发 | 调拨、报废、采购、整改审批前 | 换新预算、盘活建议、合规整改 |
| 管理层推送 | 固定接收人和周期 | 管理层摘要、部门对比 |

### 5.2 条件结构

```json
{
  "report_type": "asset_ip_budget",
  "period": {"from": "2026-06-01", "to": "2026-06-30"},
  "organization_scope": {
    "department_ids": ["uuid"],
    "site_ids": ["campus-a"]
  },
  "data_sources": ["assets", "tickets", "ipam", "licenses", "asset_audits"],
  "trigger": {
    "type": "scheduled",
    "frequency": "monthly"
  },
  "risk_thresholds": {
    "ip_conflicts": 1,
    "idle_score": 60,
    "sla_rate_below": 0.9,
    "repair_cost_over_book_value": 0.4
  },
  "recipients": ["it_manager", "finance_owner"],
  "requires_approval": true
}
```

### 5.3 生成前校验

生成前必须检查：

- 用户是否有报表查看权限。
- 是否选择统计周期和组织范围。
- 数据源是否完整，缺失数据需提示。
- 是否涉及敏感数据导出。
- AI 生成内容是否需要审批后发送。

## 6. AI 输出格式

### 6.1 标准输出结构

| 字段 | 说明 |
|------|------|
| `summary` | 100-300 字管理摘要 |
| `kpi_highlights` | KPI 亮点和异常 |
| `findings` | 关键发现列表 |
| `root_causes` | 原因解释 |
| `risk_level` | low/medium/high/critical |
| `recommendations` | 建议动作 |
| `pending_actions` | 待确认动作 |
| `data_sources` | 引用的数据来源和时间范围 |
| `export_formats` | PDF/Excel/CSV |
| `audit_record` | 生成时间、生成人、Prompt 版本、审批状态 |

### 6.2 示例输出

```json
{
  "summary": "本月 IT 资产整体运行稳定，但办公终端网段出现 3 个 IP 冲突，销售部有 6 台终端闲置评分超过 70，可优先盘活。",
  "findings": [
    "办公终端网段 IP 使用率达到 73%，存在 3 个冲突地址",
    "12 台终端闲置评分超过 60，预计可节省采购预算 8.6 万元",
    "Adobe Acrobat Pro 有 4 个授权 60 天未使用"
  ],
  "risk_level": "medium",
  "recommendations": [
    "回收 11 个 45 天未出现的 DHCP 地址",
    "将 5 台健康度高的闲置终端加入新员工调拨池",
    "创建软件许可回收整改工单"
  ],
  "pending_actions": [
    {"type": "create_ticket", "label": "创建 IP 冲突处理工单"},
    {"type": "submit_approval", "label": "提交资产调拨审批"},
    {"type": "export_report", "label": "导出月度 PDF 报表"}
  ]
}
```

## 7. API 草案

> 本节为后续实现参考，不要求第一阶段开发真实后端。

获取可用报表模板。

## 8. 审批与安全边界

AI 报表可以自动生成：

- 管理摘要。
- 指标解释。
- 异常发现。
- 风险等级。
- 建议动作。
- 待确认动作草稿。

以下动作必须人工确认：

- 分配或释放 IP。
- 安装、卸载或回收软件许可。
- 调拨资产。
- 报废或换新。
- 提交采购/预算审批。
- 将报表发送给管理层或外部邮箱。

安全要求：

- 报表导出记录审计日志。
- 涉及个人、财务、合规数据的报表需 RBAC 控制。
- AI 输出必须附数据来源和统计周期。
- AI 不得生成绕过授权、隐藏违规、删除审计证据的建议。

## 9. 页面交互验收

| 场景 | 预期 |
|------|------|
| 切换报表页签 | KPI、图表、明细和 AI 结论同步变化 |
| 修改统计周期 | 所有指标重新计算，AI 结论标注新周期 |
| 点击 AI 生成 | 展示生成条件确认页，再生成报表 |
| 异常触发 | 自动生成风险摘要，但高风险动作保持待确认 |
| 导出报表 | 支持 PDF/Excel/CSV，并记录审计 |
| 查看数据来源 | 可以追溯到资产、工单、IPAM、许可或盘点差异 |

## 10. 第一阶段落地建议

第一阶段建议以静态样例和规则模拟完成：

1. 报表模板固定为 6 类。
2. AI 结论使用规则拼装，保留 LLM 接入点。
3. 生成条件先存为 JSON 草案，不接真实调度。
4. 待确认动作只展示按钮和提示，不执行真实操作。
5. 后续再接入 Celery Beat、报表快照表、PDF 导出和钉钉推送。

## 1. 功能全景图

### 1.1 系统定位

IntelliServe IT Suite 面向 2000-5000 台终端规模的大型园区 IT 运维团队，提供从被动响应到主动预防的一站式智能运维平台。支持多 VLAN 分区部署，以钉钉为首要 IM 入口。

## 2. 核心功能模块

### 2.1 F01 仪表盘

**用户故事**：作为 IT 运维经理，我希望登录系统后立即看到关键运维指标概览，以便快速了解当前运维态势。

| 子功能 | 描述 | 优先级 |
|--------|------|--------|
| F01.1 KPI 概览卡片 | 展示资产总数、待处理工单数、AI 诊断次数、自动化执行数，支持同比/环比变化 | P0 |
| F01.2 监控趋势图表 | CPU/内存使用趋势折线图，支持 24h/7天/30天 切换 | P0 |
| F01.3 告警分布统计 | 环形图展示严重/警告/信息/已解决告警分布 | P0 |
| F01.4 最近工单列表 | 展示最近 5 条工单，含状态、优先级、提交时间 | P0 |
| F01.5 资产健康状态 | 进度条展示在线/告警/离线/闲置设备占比 | P0 |
| F01.6 AI 聊天助手入口 | 内嵌 AI 助手对话面板，支持快速提问 | P1 |

### 2.2 F02 资产管理

**用户故事**：作为 IT 管理员，我需要管理所有 IT 资产的全生命周期，从入库到报废。

| 子功能 | 描述 | 优先级 |
|--------|------|--------|
| F02.1 资产台账 CRUD | 资产编号、设备名称、型号、部门、状态、SN、采购日期等字段管理 | P0 |
| F02.2 OCR 自动录入 | 通过 PaddleOCR 识别设备标签照片，自动提取 SN、型号、制造商 | P1 |
| F02.3 资产状态监控 | 集成 Zabbix Agent 数据，实时展示 CPU/内存/磁盘使用率 | P0 |
| F02.4 健康度评估 | 基于监控指标的综合健康评分（0-100） | P2 |
| F02.5 闲置设备识别 | 超过 N 天无心跳或无用户登录的设备标记为闲置 | P1 |
| F02.6 资产分配管理 | 资产-用户分配关系管理，支持分配/回收操作 | P1 |
| F02.7 批量导入导出 | CSV/Excel 批量导入资产数据，导出资产报表 | P1 |

### 2.3 F03 工单管理

**用户故事**：作为 IT 工程师，我需要高效地处理用户提交的 IT 问题工单。

| 子功能 | 描述 | 优先级 |
|--------|------|--------|
| F03.1 工单创建 | 支持手动创建、AI 聊天自动创建、监控告警自动创建 | P0 |
| F03.2 状态流转 | 看板视图：待处理 → AI 诊断中 → 处理中 → 已解决 → 已关闭 | P0 |
| F03.3 AI 诊断集成 | 点击"AI 诊断"按钮，异步调用 LLM 分析故障并给出建议 | P0 |
| F03.4 工程师指派 | 手动指派或 AI 推荐（基于技能标签 + 当前负载） | P1 |
| F03.5 解决方案记录 | 记录故障原因、解决方案、使用的脚本 | P0 |
| F03.6 工单提取至知识库 | 一键将已解决工单转换为知识库文章 | P1 |

### 2.4 F05 智能聊天机器人

**用户故事**：作为终端用户，我希望通过企业微信/钉钉直接描述问题，获得即时帮助。

| 子功能 | 描述 | 优先级 |
|--------|------|--------|
| F05.1 企业微信集成 | 接收企微 Bot 消息回调，格式化回复 | P0 |
| F05.2 钉钉集成 | 通过 Stream Mode 接收钉钉消息 | P1 |
| F05.3 意图分类路由 | LLM 判断意图：问候/知识查询/故障报告/服务请求 | P0 |
| F05.4 RAG 知识问答 | 知识库语义检索 + LLM 生成回答 | P0 |
| F05.5 自动化触发 | L1 故障自动匹配修复脚本并执行 | P0 |
| F05.6 工单创建升级 | 无法自动解决时创建工单并指派工程师 | P0 |

### 2.5 F06 自动化引擎

**用户故事**：作为 IT 工程师，我需要将重复性的故障修复操作自动化。

| 子功能 | 描述 | 优先级 |
|--------|------|--------|
| F06.1 脚本模板管理 | PowerShell 脚本的版本化管理，含描述、风险等级、参数定义 | P0 |
| F06.2 远程执行 | 通过 WinRM/PowerShell Remoting 在目标终端执行脚本 | P0 |
| F06.3 风险等级审批 | Low 自动执行、Medium 用户确认、High 工程师审批 | P0 |
| F06.4 执行日志记录 | 记录执行参数、输出、耗时、结果（成功/失败） | P0 |
| F06.5 执行结果回调 | 脚本执行完成后通知工单系统和用户 | P1 |

## 4. 功能交互流程

### 4.1 典型用户自助服务流程

```
用户 (企微/钉钉)
    │
    ├─ 输入: "连不上网了"
    │
    ▼
智能聊天机器人 (F05)
    │
    ├─ F05.3 意图分类 → fault_report
    ├─ F05.4 RAG 检索 → 匹配知识库文章
    │
    ├─ L1 自动修复 ──────────────────────┐
    │   ├─ F05.5 匹配脚本: network-reset.ps1
    │   ├─ F06.2 远程执行
    │   └─ 返回结果: "已重置网络，请重试"  │
    │                                      │
    ├─ L2 AI 诊断 ───────────────────────┤
    │   ├─ LLM 分析故障日志               │
    │   └─ 生成诊断报告 + 建议操作         │
    │                                      │
    └─ L3 升级工单 ───────────────────────┘
        ├─ F03.1 自动创建工单
        ├─ F03.4 AI 推荐工程师
        └─ 通知工程师接单
```

### 4.2 工单生命周期

```
创建 ──→ 待处理 ──→ AI 诊断中 ──→ 处理中 ──→ 已解决 ──→ 已关闭
  │                    │              │           │
  │                    │              │           └─→ F03.6 提取至知识库
  │                    │              │
  │                    │              └─→ 等待用户确认
  │                    │
  │                    └─→ L1 自动修复 → 直接解决
  │
  └─→ 来源: 聊天机器人 / 手动创建 / 监控告警
```

## 5. 功能约束与边界

### 5.1 功能边界

| 包含 | 不包含 |
|------|--------|
| Windows 10/11 + macOS 桌面终端管理 | Linux 终端管理（Phase 3 考虑） |
| 企业微信 + 钉钉 IM 集成 | Slack/Teams 等海外 IM |
| 开发期调用百炼 DeepSeek-V4 API，生产支持私有化 DeepSeek-V4 模型网关 | 海外 SaaS LLM 作为默认依赖 |
| Zabbix Agent 监控 | Prometheus/SNMP 全量监控 |
| PowerShell + 受控 Shell 自动化脚本 | Ansible/Puppet 配置管理（Phase 3） |
| 单站点/多 VLAN 部署 | 跨地域多站点高可用（Phase 3+） |

### 5.2 性能约束

| 指标 | 目标值 |
|------|--------|
| 页面加载时间 | < 2 秒 |
| API 响应时间 (P95) | < 500ms |
| LLM 首次响应时间 | < 5 秒 |
| 脚本执行超时 | 300 秒 |
| 并发用户数 | 50+ |
| 监控数据采集延迟 | < 60 秒 |

### 5.3 安全约束

| 约束 | 说明 |
|------|------|
| 数据不出内网 | 所有 AI 推理、数据存储均在本地完成 |
| 最小权限原则 | RBAC 三级权限，脚本分级审批 |
| 审计可追溯 | 所有 API 调用记录审计日志 |
| 传输加密 | TLS 1.3，Phase 3 升级 mTLS |
| 存储加密 | 敏感字段 AES-256-GCM 列级加密 |

## 6. 非功能需求

### 6.1 性能需求

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| 页面首屏加载时间 | < 2 秒 | Lighthouse Performance Score |
| API 响应时间 (P95) | < 500ms | Prometheus histogram |

| 并发 Agent 连接 | 5000 | Agent 心跳负载测试 |
| 监控数据采集延迟 | < 60 秒 | Agent → Server 端到端 |

### 6.2 可用性需求

| 指标 | 目标值 |
|------|--------|
| 系统整体可用性 | 99.5%（月允许宕机 < 3.65 小时） |
| RTO (恢复时间目标) | < 4 小时 |
| RPO (恢复点目标) | < 1 小时（数据库增量备份间隔） |
| LLM 服务降级可用性 | 规则引擎回退保证 100% 基础服务可用 |

### 6.3 可扩展性需求

| 维度 | 目标 |
|------|------|
| 终端规模 | 2000-5000 台（Phase 1-2），可扩展至 10000+（Phase 3+） |

| 并发工单 | 支持 500+ 工单/天峰值处理 |
| AI 推理并发 | 单 GPU 2 并发，支持 GPU 扩展 |

### 6.4 可维护性需求

| 指标 | 目标值 |
|------|--------|
| 部署时间 | < 30 分钟（一键部署） |
| Agent 升级时间 | < 5 分钟/终端（静默升级） |
| 知识库更新生效 | < 1 分钟（重新嵌入后） |

## 8. F02 资产管理增强：成本、IP/MAC、软件与外设

### 8.1 资产类别扩展

| 类别 | 典型对象 | 纳管重点 |
|------|----------|----------|
| 终端 | 笔记本、台式机、瘦客户端、移动终端 | SN、MAC、IP、使用人、健康度、软件清单、成本 |
| 网络设备 | 交换机、路由器、防火墙、无线控制器、无线 AP | 管理 IP、端口、SNMP、固件、配置备份 |
| 无线设备 | AC、AP、SSID、射频、信道 | 覆盖区域、在线客户端、干扰、漫游质量 |
| 外设 | 打印机、扫描仪、会议屏、摄像头、门禁、UPS | IP、驱动、耗材、位置、故障脚本 |
| 软件资产 | 软件库、安装记录、许可证、订阅 | 标准软件、安装包、授权、合规、卸载策略 |

### 8.2 资产成本字段

资产详情和导入模板新增：

- 购入金额、购入日期、供应商、发票号、采购单号。
- 折旧方法、折旧周期、残值率、账面净值、估算残值。
- 累计维修成本、最近维修成本、保修到期日、维保合同。
- 是否闲置、使用率、闲置评分、盘活建议。

### 8.3 闲置盘活流程

1. Agent/网络扫描采集在线、登录、性能和网络活动。
2. 系统计算闲置评分，并按部门、型号、健康度筛选可盘活资产。
3. AI 结合新员工设备需求、资产健康度和账面净值生成调拨建议。
4. 管理员确认后生成调拨单，更新使用人、部门、位置、IP 绑定。
5. 报表记录节省预算、释放软件许可和 IP 资源。

### 8.4 维修、报废与换新建议

| 条件 | 系统建议 |
|------|----------|
| 保修期内、维修成本低 | 创建维修工单 |
| 累计维修成本超过账面净值 40% | 建议换新评估 |
| 超过折旧周期且故障频发 | 建议报废/换新审批 |
| 健康度高但长期闲置 | 建议调拨 |
| 资产低使用且占用高价值软件 | 建议回收软件许可 |

## 9. 新增功能模块

### 9.1 IPAM 地址管理

| 子功能 | 说明 |

## 1. 设计目标

资产管理不只记录“有什么设备”，还要回答三个管理问题：

| 问题 | 系统能力 |
|------|---------|
| 资产在哪里、谁在用、联网状态如何 | 台账、Agent 心跳、MAC/IP 绑定、盘点差异 |
| 资产值多少钱、还值多少钱、还该不该修 | 采购成本、折旧、账面净值、维修成本、残值估算 |
| 哪些资产闲置、能不能调拨、是否该报废/换新 | 闲置评分、调拨建议、报废审批、预算报表 |

第一阶段采用规则估算和静态阈值，不接外部二手市场 API；后续可对接财务系统、采购系统和二手残值数据源。

## 2. 资产财务字段

### 2.1 核心字段

| 字段 | 说明 |
|------|------|
| `purchase_cost` | 购入金额，含税口径需在系统参数中统一 |
| `purchase_date` | 购入日期，用于折旧起算 |
| `supplier_name` | 供应商 |
| `invoice_no` | 发票号或采购单号 |
| `warranty_expiry` | 保修到期日 |
| `depreciation_method` | 折旧方法，第一版支持 `straight_line` |
| `depreciation_months` | 折旧周期，例如笔记本 36 个月、服务器 60 个月 |
| `salvage_rate` | 预计残值率，例如 5%-15% |
| `book_value` | 当前账面净值 |
| `market_residual_value` | 估算市场残值，第一版由规则配置 |
| `repair_cost_total` | 累计维修成本 |
| `last_repair_cost` | 最近一次维修成本 |
| `refresh_recommendation` | 保留、调拨、维修、换新、报废 |

### 2.2 折旧规则

第一版采用直线折旧：

```text
月折旧额 = 购入金额 * (1 - 残值率) / 折旧月数
已折旧月数 = min(当前月份 - 购入月份, 折旧月数)
账面净值 = max(购入金额 - 月折旧额 * 已折旧月数, 购入金额 * 残值率)
```

默认规则建议：

| 资产类型 | 折旧周期 | 残值率 | 换新观察点 |
|----------|----------|--------|------------|
| 笔记本/台式机 | 36 个月 | 8% | 30 个月后进入换新评估 |
| 服务器 | 60 个月 | 10% | 48 个月后进入容量与维保评估 |
| 网络设备 | 60 个月 | 10% | 固件 EOL 或维保到期优先 |
| 打印机/扫描仪 | 48 个月 | 5% | 维修成本超过净值 40% |
| 会议屏/摄像头/门禁 | 48 个月 | 8% | 兼容性与故障率优先 |

## 3. 闲置识别与盘活评分

### 3.1 识别信号

| 信号 | 来源 | 说明 |
|------|------|------|
| Agent 心跳 | 终端 Agent / Zabbix | 连续离线天数、在线时长 |
| 登录使用 | AD/本地登录日志 | 最近登录用户、登录频率 |
| 性能活动 | CPU、内存、磁盘、进程 | 判断是否真实使用 |
| 网络活动 | DHCP、ARP、交换机 MAC 表 | 判断设备是否接入网络 |
| 工单记录 | 工单模块 | 频繁故障设备降低再分配优先级 |
| 软件使用 | 软件审计 | 是否占用高价值许可证 |

### 3.2 闲置评分

```text
idle_score =
  35% * 离线天数评分 +
  25% * 登录频率评分 +
  20% * 性能活动低使用评分 +
  10% * 网络活动评分 +
  10% * 人工状态评分
```

评分分级：

| 分数 | 状态 | 建议动作 |
|------|------|----------|
| 0-39 | 正常使用 | 保持观察 |
| 40-59 | 低利用 | 提醒部门确认用途 |
| 60-79 | 疑似闲置 | 推荐调拨、回收或释放软件许可 |
| 80-100 | 高闲置 | 进入盘活清单，生成审批任务 |

## 4. 维修 vs 换新判断

系统在资产详情页给出 AI 建议，但最终动作必须由管理员确认。

| 判断条件 | 建议 |
|----------|------|
| 保修期内且维修成本为 0 | 优先维修 |
| 累计维修成本 > 账面净值 40% | 建议换新或报废 |
| 近 90 天故障工单 >= 3 次 | 建议换新评估 |
| 资产已超过折旧周期且性能评分低于 60 | 建议换新 |
| 资产闲置但健康度高于 75 | 建议调拨 |
| 软件许可价值高且设备低使用 | 建议回收许可或迁移许可 |

AI 输出格式：

| 字段 | 示例 |
|------|------|
| 建议 | “调拨给市场部新员工” |
| 依据 | “近 30 天在线 2 小时，账面净值 1,280 元，健康度 86” |
| 影响 | “释放一台新购预算，节省约 6,800 元” |
| 待确认动作 | “提交调拨审批” |

## 5. IP/MAC 与 IPAM 联动

资产必须支持多网卡、多 MAC、多 IP 的绑定历史。

| 场景 | 设计 |
|------|------|
| 终端入库 | 记录有线 MAC、无线 MAC、当前 DHCP IP |
| 新员工分配 | 系统从 IP 地址池推荐空闲 IP，并绑定资产/用户 |
| IP 冲突 | DHCP/ARP/Agent 上报比对，识别重复 IP 或重复 MAC |
| 保留地址 | 服务器、打印机、门禁、会议设备可设静态保留 |
| 回收资产 | 释放 IP、保留历史绑定记录 |

报表应展示：地址池使用率、空闲 IP、冲突 IP、保留地址、长期未使用地址。

## 6. 软件库、打印机与外设盘活

### 6.1 软件库

软件库统一管理标准软件、安装包、版本、静默安装参数、卸载命令和许可策略。

| 能力 | 说明 |
|------|------|
| 标准软件目录 | 区分必装、可选、受控、禁用软件 |
| 安装包管理 | 存储下载地址、哈希、适配系统 |
| 静默参数 | 例如 MSI 参数、EXE 静默参数 |
| 授权策略 | 按用户、设备、部门或并发许可 |
| 安装分布 | 查看软件在哪些资产上安装 |
| AI 建议 | 推荐卸载闲置/违规软件，释放许可 |

### 6.2 打印机与驱动库

打印机作为资产和服务入口双重管理。

| 能力 | 说明 |
|------|------|
| 打印机台账 | 型号、IP、MAC、位置、部门、耗材、状态 |
| 驱动库 | 驱动包、适配系统、版本、签名状态 |
| 安装命令 | 静默安装、端口创建、默认打印机配置 |
| 故障脚本 | 重启 Spooler、清理队列、重建 TCP/IP Port |
| AI 匹配 | 根据系统版本和打印机型号推荐驱动 |

## 7. 自动盘点与账实差异

自动盘点任务按日或按周执行：

1. Agent 上报终端硬件、MAC、IP、软件、登录用户。
2. 网络扫描补充未安装 Agent 的设备，如交换机、AP、打印机、摄像头。
3. 系统与资产台账比对，生成差异：
   - 台账存在但网络未发现。
   - 网络发现但台账不存在。
   - MAC/IP 与台账不一致。
   - 使用人/部门与实际登录不一致。
   - 软件安装与授权策略不一致。
4. AI 生成处理建议并创建待确认动作。

差异处理必须留下审计记录：发现时间、证据、建议、处理人、处理结果。

## 8. 沙箱演示场景

沙箱用于向业务方演示“AI 建议 + 解释 + 待确认动作”的闭环，不直接操作真实终端。

| 场景 | AI 诊断 | 待确认动作 |
|------|---------|------------|
| 新员工设备 + IP 分配 | 推荐闲置高健康笔记本和空闲 IP | 确认调拨、确认分配 IP |
| 会议室 Wi-Fi 异常 | AP 信道拥塞或弱信号 | 推送巡检工单或调整信道 |
| 打印机驱动故障 | 驱动版本不匹配或队列堵塞 | 推送驱动、重启打印服务 |
| 闲置资产盘活 | 低使用但健康度高 | 提交调拨审批 |
| 未授权软件整改 | 检测违规安装或许可超额 | 卸载软件、回收许可 |
| 旧设备维修/换新 | 维修成本高于阈值 | 提交换新/报废审批 |

## 9. 报表清单

| 报表 | 关键指标 |
|------|----------|
| 资产成本报表 | 原值、净值、折旧、残值、维修成本 |
| 闲置盘活报表 | 闲置数量、可调拨资产、节省预算 |
| IP 分配报表 | 地址池使用率、冲突、保留、空闲 |
| 自动盘点差异报表 | 账实不符数量、差异类型、处理状态 |
| 软件合规报表 | 超额许可、违规软件、闲置许可 |
| 换新预算报表 | 建议换新设备、预算、优先级 |

## 4. 组件规范

### 4.1 按钮 (Button)

| 类型 | 类名 | 用途 |
|------|------|------|
| 主按钮 | `.btn-primary` | 主要操作（创建、提交） |
| 次按钮 | `.btn-secondary` | 次要操作（取消、返回） |
| 幽灵按钮 | `.btn-ghost` | 辅助操作（筛选、切换） |
| 小按钮 | `.btn-sm` | 表格行内、紧凑区域 |
| 图标按钮 | `.btn-icon` | 纯图标操作（通知、菜单） |

### 4.2 卡片 (Card)

| 类名 | 用途 |
|------|------|
| `.card` | 默认卡片，hover 边框变化 |

## 7. 无障碍访问规范

### 7.1 目标

遵循 WCAG 2.1 AA 等级标准，确保系统对各类用户可用。

| 要求 | 实现 |
|------|------|
| 键盘导航 | 所有交互元素可通过 Tab/Shift+Tab 导航，Enter/Space 激活 |
| 焦点指示 | 所有可聚焦元素有可见的焦点环（2px #3b82f6） |
| 颜色对比度 | 正文文字与背景对比度 ≥ 4.5:1，大文字 ≥ 3:1 |
| 非颜色依赖 | 状态指示同时使用颜色和文字/图标（如状态圆点 + 文字标签） |
| 屏幕阅读器 | 图标按钮添加 `aria-label`，图表提供文本摘要 |
| 表单标签 | 所有输入框关联 `<label>` 或 `aria-labelledby` |
| 跳过导航 | 提供"跳到主内容"链接 |
| 缩放支持 | 支持 200% 缩放不丢失功能 |

### 7.2 实现检查清单

- [ ] 使用 `<button>` 而非 `<div onclick>` 作为可交互元素
- [ ] 表格使用 `<thead>/<tbody>` 结构
- [ ] Modal 打开时焦点移入，关闭时焦点返回触发元素
- [ ] Toast/通知使用 `role="alert"` 或 `aria-live`
- [ ] 图表提供等价的表格数据视图

## 8. 国际化设计（i18n 预留）

### 8.1 当前范围

Phase 1-2 仅支持简体中文。以下为国际化扩展预留的设计决策：

| 层面 | 预留策略 |
|------|---------|
| 文案管理 | 所有用户可见文案通过 i18n key 引用，集中管理 |
| 日期格式 | 使用 `dayjs` 的 locale 系统，非硬编码格式 |
| 数字格式 | 千分位分隔符可配置 |
| 布局方向 | CSS 使用逻辑属性（`margin-inline-start` 代替 `margin-left`） |
| 字体 | Inter（拉丁）+ Noto Sans SC（中文），可扩展其他 Noto 系列 |
| 图标 | 优先使用 Ant Design Icons（无文字依赖） |
| 技术选型 | 未来 Phase 3 可集成 `react-intl` 或 `i18next` |

## 9. 资产成本、IPAM、软件库与沙箱原型补充

| 表格列 | 资产编号、设备名称、型号、部门、状态、MAC、IP、购入金额、购入日期、账面净值、使用率、闲置风险 |
| 快捷操作 | OCR 录入、添加资产、导入成本、导出盘点差异 |

### 9.2 资产详情抽屉

资产详情抽屉新增以下卡片：

| 卡片 | 内容 |
|------|------|

| 折旧曲线 | 原值、已折旧、当前净值、预计残值 |

| 软件清单 | 标准软件、受控软件、未授权软件、最近使用 |

### 9.3 IPAM 页面

页面布局：

- 顶部 KPI：地址池数量、已分配、空闲、冲突、保留。
- 左侧地址池列表：网段、VLAN、用途、使用率。
- 右侧分配表：IP、状态、MAC、资产、使用人、最近发现。
- 推荐分配卡：输入部门/资产后展示推荐 IP、原因和“确认分配”按钮。

### 9.4 软件库页面

页面布局：

- 软件库表格：软件名、版本、标准级别、许可策略、安装包、静默参数。
- 安装分布卡：安装数量、合规数量、未授权数量、闲置许可。
- AI 整改建议：推荐卸载目标、可回收许可、需采购数量。

### 9.5 打印机与驱动库页面

页面布局：

- 打印机台账：名称、型号、IP、MAC、位置、状态、推荐驱动。
- 驱动库：厂商、适配型号、系统版本、驱动版本、安装命令。
- 故障处理：清理队列、重启 Spooler、推送驱动、生成工单。

### 9.6 报表页面

报表页新增卡片：

| 报表 | 页面呈现 |
|------|----------|
| 资产盘点差异 | 差异类型柱状图 + 待处理列表 |
| 闲置资产盘活 | 闲置风险排行 + 预计节省预算 |
| IP 分配 | 地址池使用率 + 冲突列表 |
| 软件合规 | 超额许可、违规软件、闲置许可 |
| 换新预算 | 建议换新清单 + 月度预算 |

### 9.7 沙箱演示任务页

沙箱页采用“任务卡 + AI 诊断 + 待确认动作”的三段结构：

1. 选择演示任务。
2. 展示 AI 诊断依据和影响。
3. 点击确认按钮，模拟生成审批、工单或配置动作。

沙箱任务包括：

- 新员工设备 + IP 分配。
- 会议室 Wi-Fi 异常排查。
- 打印机驱动故障修复。
- 闲置资产盘活建议。
- 未授权软件合规整改。
- 旧设备维修/换新预算判断。
